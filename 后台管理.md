# 权限与身份认证模块
这一部分最重要的是一份好的数据设计，参考文件`permission.ts`

有了它，我们就需要一份对这份数据进行处理的工具，将用户可访问菜单转换为路由配置，校验用户是否拥有目标权限，构建接口请求参数等等，参考`PermissionUtil.ts`

- 用户认证工具：管理用户登录状态（持久化存储、读取、销毁），处理令牌有效性。
- 权限校验工具：前端快速判断用户是否拥有指定权限（按钮、接口、菜单），支撑权限控制逻辑。
- 菜单处理工具：将后端返回的 menus 数据，转换为前端路由 / 菜单组件可用的格式。
- 数据权限工具：提取用户数据访问范围，支撑前端数据筛选 / 接口参数携带。
- 角色工具：便捷操作用户角色数据，支撑基于角色的权限控制（RBAC）。
- 整体权限工具：整合所有权限数据，提供一站式取值 / 初始化能力。
